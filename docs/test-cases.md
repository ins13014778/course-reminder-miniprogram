# 测试用例文档

最后更新：2026-03-19

这份文档面向后续开发者、测试人员和 AI，目标是给当前项目提供一套可重复执行的回归用例。重点覆盖高风险链路：登录、课表导入、分享密钥、提醒、公告、后台治理和权限封禁。

## 1. 测试范围

当前建议覆盖的模块：

- 小程序登录
- 用户资料与首页公告
- 课表导入与课表管理
- 分享密钥生成 / 导入
- 笔记发布 / 编辑 / 删除 / 审核
- 订阅提醒
- 公告发布
- 后台权限治理

## 3. 冒烟测试

每次大改后至少执行以下冒烟测试：

1. 后端 `npm run build` 成功
2. 后台 `npm run build` 成功
3. 后台首页能打开
4. 后台用户页能看到用户列表
5. 后台笔记页能看到笔记内容
6. 小程序能登录
7. 小程序首页能读到公告

## 4. 功能测试用例

## 4.1 登录模块

### TC-LOGIN-001 正常登录

前置条件：

- 微信开发者工具可正常打开小程序
- CloudBase 环境可用

步骤：

1. 打开小程序登录页
2. 点击微信登录
3. 同意用户信息授权

预期结果：

- 登录成功
- 本地缓存写入 `token` 和 `user`
- 跳回上一页或首页
- `users` 表存在该用户记录

### TC-LOGIN-002 账号被后台封禁后登录

前置条件：

- 后台已将某测试用户账号权限设为封禁

步骤：

1. 使用该测试用户重新进入登录页
2. 点击微信登录

预期结果：

- 登录失败
- 用户可看到明确封禁提示
- 不应成功进入已登录状态

### TC-LOGIN-003 临时封禁过期后恢复登录

前置条件：

- 某测试用户为“限时封禁”
- 封禁结束时间已过

步骤：

1. 使用该用户重新登录

预期结果：

- 登录恢复正常
- 不再提示封禁

## 4.2 公告模块

### TC-ANN-001 后台发布公告，前台可见

步骤：

1. 在后台公告页新建一条公告
2. 设置为 `published`
3. 刷新小程序首页

预期结果：

- 小程序首页显示最新已发布公告
- 标题和正文内容正确

### TC-ANN-002 草稿公告前台不可见

步骤：

1. 在后台创建草稿公告
2. 不发布
3. 刷新小程序首页

预期结果：

- 草稿公告不会出现在首页

### TC-ANN-003 删除公告后前台同步消失

步骤：

1. 删除一条已发布公告
2. 刷新首页

预期结果：

- 该公告不再显示

### TC-COURSE-001 默认模板导入

步骤：

1. 登录小程序
2. 进入导入页
3. 执行默认模板导入

预期结果：

- 用户课表写入数据库
- 课程页可见导入结果

### TC-COURSE-002 分享密钥导入

步骤：

### TC-COURSE-003 后台删除异常课程

步骤：

1. 在后台课表巡检页找到一条课程

预期结果：

- 用户端课表同步减少该课程

## 4.4 分享密钥模块

### TC-SHARE-001 正常生成分享密钥

步骤：

1. 登录小程序
2. 打开设置页
3. 点击生成分享密钥

预期结果：

- 成功生成密钥
- `schedule_share_keys` 有记录
- 设置页显示该密钥

### TC-SHARE-002 后台禁用单个分享密钥

步骤：

1. 在后台分享密钥页找到一个可用密钥
2. 点击禁用
3. 填写原因

预期结果：

- 该密钥状态变成 `blocked`
- `is_active = 0`
- 后续用此密钥导入时失败

### TC-SHARE-003 用户分享权限封禁

步骤：

1. 在后台用户页打开某用户详情
2. 将分享密钥权限设为封禁
3. 回到小程序设置页尝试生成密钥

预期结果：

- 生成失败
- 返回明确封禁提示

## 4.7 资料违规与申诉

### TC-PROFILE-001 头像被封禁后禁止修改

步骤：
1. 在后台用户详情中将某个用户的“头像权限”设置为封禁
2. 回到小程序个人页，点击“换头像”或“自定义头像”

预期结果：
- 不进入正常上传保存流程，或保存时被明确拦截
- 用户看到“头像功能受限”的提示
- 可跳转到申诉中心

### TC-PROFILE-002 个性签名被封禁后禁止修改

步骤：
1. 在后台用户详情中将某个用户的“个签权限”设置为封禁
2. 回到小程序个人页，点击“个性签名”

预期结果：
- 不允许继续修改
- 用户看到“个性签名功能受限”的提示
- 可跳转到申诉中心

### TC-APPEAL-001 头像申诉创建成功

前置条件：
- 当前用户头像权限处于封禁状态

步骤：
1. 打开小程序申诉中心
2. 选择“头像申诉”
3. 填写标题、说明并提交

预期结果：
- `user_appeals` 新增一条 `appeal_type = 'avatar'` 的待处理记录
- 小程序提示提交成功

### TC-APPEAL-002 个签申诉审核通过后自动解封

前置条件：
- 存在一条 `appeal_type = 'signature'` 的待处理申诉
- 对应用户 `signature_status = 'banned'`

步骤：
1. 在后台申诉页打开该条记录
2. 输入 `approved`
3. 提交后台备注

预期结果：
- `user_appeals.status = 'approved'`
- `users.signature_status = 'active'`
- `users.signature_ban_reason = NULL`
- `users.signature_banned_until = NULL`

步骤：

1. 等待提醒调度执行
2. 查询 `reminders` 表

预期结果：

- 生成新的提醒记录
- 状态为 `pending` 或发送后为 `sent`

## 4.7 后台治理模块

### TC-ADMIN-001 打开用户详情

步骤：

1. 后台进入用户治理页
2. 选择任意用户点击“查看详情”

预期结果：

- 抽屉打开
- 可见用户信息、完整课表、笔记、分享密钥、订阅信息

### TC-ADMIN-002 账号临时封禁

步骤：

1. 在用户详情中将账号状态改为封禁
2. 设置封禁天数
3. 填写原因并保存

预期结果：

- `users.account_status = 'banned'`
- `account_banned_until` 写入
- 登录被拦截

### TC-ADMIN-003 账号永久封禁

步骤：

1. 在用户详情中将账号状态改为封禁
2. 勾选永久封禁
3. 保存

预期结果：

- `account_banned_until = null`
- 登录持续被拦截

### TC-ADMIN-004 解除账号封禁

步骤：

1. 在用户详情中把账号状态改回正常
2. 保存

预期结果：

- `account_status = 'active'`
- 原因和到期时间清空

### TC-ADMIN-005 笔记权限临时封禁

步骤：

1. 在用户详情将笔记权限设为封禁
2. 设置天数
3. 保存

预期结果：

- 用户无法新增、编辑、删除笔记

### TC-ADMIN-006 分享密钥权限永久封禁

步骤：

1. 在用户详情将分享密钥权限设为永久封禁
2. 保存

预期结果：

- 用户无法生成分享密钥
- 相关 SQL 操作被云函数拦截

## 8. 最后建议

这个项目的测试重点不是“页面能不能打开”，而是“真实链路是否被拦住或放通”。

尤其以下几类一定要测真实结果：

- 账号封禁后还能不能登录
- 笔记权限封禁后还能不能写笔记
- 分享密钥权限封禁后还能不能生成或导入
- 后台下架笔记后，用户端是否真的看不到
- 公告发布后，小程序首页是否真的能看见
- 提醒授权写入后，后台列表和数据库是否一致
