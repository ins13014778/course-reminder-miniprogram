# 测试用例文档

最后更新：`2026-03-20`

这份文档面向后续开发者、测试人员和 AI，目标是给当前项目提供一套可重复执行的回归用例。重点覆盖高风险链路：登录、课表、分享、笔记治理、提醒、公告、申诉、反馈、批量治理和高风险确认。

## 1. 测试范围

当前建议覆盖：

- 小程序登录
- 首页公告与通知中心
- 课表导入与课表管理
- 分享密钥生成 / 导入 / 封禁
- 笔记发布 / 编辑 / 删除 / 审核
- 举报提交与审核
- 申诉提交与审核
- 留言反馈提交与审核
- 提醒订阅、提醒日志、失败重试
- 后台权限治理与管理员二次确认

## 3. 冒烟测试

每次大改后至少执行以下冒烟测试：

1. 后端 `npm run build` 成功
2. 后台 `npm run build` 成功
3. 后台概览页能打开
4. 后台用户页能看到用户列表
5. 后台笔记页能看到笔记内容
6. 后台提醒日志页能打开
7. 小程序能登录
8. 小程序首页能读到公告
9. 小程序通知中心能打开

## 4. 功能测试用例

### 4.1 登录模块

#### TC-LOGIN-001 正常登录

步骤：

1. 打开小程序登录页
2. 点击微信登录
3. 同意用户信息授权

预期结果：

- 登录成功
- 本地缓存写入 `token` 和 `user`
- 跳回首页或上一页
- `users` 表存在该用户记录

#### TC-LOGIN-002 账号被封禁后登录

前置条件：

- 后台已将某测试用户账号设为封禁

步骤：

1. 使用该用户重新登录

预期结果：

- 登录失败
- 用户能看到明确封禁提示

### 4.2 公告与通知中心

#### TC-ANN-001 后台发布公告后首页可见

步骤：

1. 在后台公告页新建一条公告
2. 状态设置为 `published`
3. 刷新小程序首页

预期结果：

- 首页显示最新公告
- 标题和正文正确

#### TC-MSG-001 通知中心合并展示公告、反馈、申诉

前置条件：

- 至少存在一条公告
- 至少存在一条已审核反馈
- 至少存在一条已审核申诉

步骤：

1. 打开小程序通知中心

预期结果：

- 列表按时间倒序展示
- 能看到公告、反馈、申诉三类消息

#### TC-MSG-002 通知中心已读状态持久化

步骤：

1. 打开通知中心
2. 点击任意一条未读消息
3. 返回列表并刷新页面

预期结果：

- 已读状态保留
- 未读数量减少

### 4.3 课表与分享

#### TC-COURSE-001 模板课表导入

步骤：

1. 登录小程序
2. 打开导入页
3. 执行模板导入

预期结果：

- 课程写入数据库
- 课表页可见导入结果

#### TC-SHARE-001 正常生成分享密钥

步骤：

1. 打开设置页
2. 点击生成分享密钥

预期结果：

- 成功生成密钥
- `schedule_share_keys` 中有记录

#### TC-SHARE-002 后台封禁单个分享密钥

步骤：

1. 在后台分享密钥页找到一条可用密钥
2. 点击禁用并填写原因

预期结果：

- 该密钥状态变为 `blocked`
- 后续使用该密钥导入失败

### 4.4 笔记与举报

#### TC-NOTE-001 正常发布笔记

步骤：

1. 登录小程序
2. 新建并发布一条笔记

预期结果：

- 发布成功
- `notes.status = 'visible'`

#### TC-NOTE-002 后台单条下架笔记

步骤：

1. 在后台笔记页定位目标笔记
2. 点击下架并填写原因

预期结果：

- 笔记状态变为 `blocked`
- 用户端不可见

#### TC-REPORT-001 举报通过并联动下架

步骤：

1. 用户提交一条举报
2. 后台举报页审核通过，并选择下架原笔记

预期结果：

- 举报状态更新
- 原笔记被联动下架

### 4.5 申诉与反馈

#### TC-APPEAL-001 用户提交申诉

前置条件：

- 用户存在被限制的账号、笔记、分享、头像或个签能力

步骤：

1. 打开小程序申诉中心
2. 选择对应申诉类型
3. 填写说明并提交

预期结果：

- `user_appeals` 生成待处理记录
- 用户看到提交成功提示

#### TC-APPEAL-002 后台审核通过后自动解封

步骤：

1. 在后台申诉页打开待审核记录
2. 输入 `approved`
3. 提交审核备注

预期结果：

- 申诉状态改为 `approved`
- 对应限制被解除
- 通知中心出现审核结果

#### TC-FEEDBACK-001 用户提交反馈，后台可审核

步骤：

1. 用户登录后提交反馈
2. 后台反馈页查看并审核

预期结果：

- `user_feedback` 生成记录
- 后台可看到内容并更新状态

### 4.6 批量治理与高风险确认

#### TC-BATCH-001 批量封禁用户权限

步骤：

1. 在后台用户页勾选多个用户
2. 点击批量权限
3. 选择批量封禁
4. 输入二次确认码

预期结果：

- 多个用户权限批量更新成功
- `user_violation_records` 生成对应记录

#### TC-BATCH-002 批量下架笔记

步骤：

1. 在后台笔记页勾选多条笔记
2. 点击批量下架
3. 输入二次确认码

预期结果：

- 多条笔记状态更新为 `blocked`
- 前台不可见

#### TC-BATCH-003 批量处理举报

步骤：

1. 在后台举报页勾选多条举报
2. 执行批量通过或批量驳回

预期结果：

- 举报状态批量更新
- 需要联动下架时同步生效

#### TC-RISK-001 高风险挑战码失效

步骤：

1. 创建高风险挑战
2. 等待其过期，或输入错误验证码
3. 执行高风险操作

预期结果：

- 操作被拒绝
- 后台提示确认码无效或已过期

### 4.7 提醒日志与重试

#### TC-REMIND-001 打开提醒订阅

步骤：

1. 登录小程序
2. 打开设置页
3. 开启提醒并完成订阅授权

预期结果：

- `user_subscriptions` 写入或更新为启用状态

#### TC-REMIND-002 后台查看提醒汇总

步骤：

1. 进入后台提醒日志页

预期结果：

- 可以看到汇总卡片
- 可以看到失败日志和重试次数

#### TC-REMIND-003 手动重试失败日志

前置条件：

- `reminder_send_logs` 中存在失败记录

步骤：

1. 在后台提醒日志页勾选失败记录
2. 点击重试

预期结果：

- 生成新的提醒日志记录
- 原记录 `retry_count` 增加
- `last_retry_at` 更新

## 5. 回归重点

每次改动以下模块后，建议至少回归对应用例：

- 改登录：回归 `TC-LOGIN-*`
- 改公告或消息中心：回归 `TC-ANN-*` 和 `TC-MSG-*`
- 改笔记或举报：回归 `TC-NOTE-*` 和 `TC-REPORT-*`
- 改申诉或反馈：回归 `TC-APPEAL-*` 和 `TC-FEEDBACK-*`
- 改后台治理：回归 `TC-BATCH-*` 和 `TC-RISK-*`
- 改提醒：回归 `TC-REMIND-*`
